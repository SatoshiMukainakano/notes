JSなどでDOMを書き換えるとき、、、
* aria-busy : 更新前＝false、更新中＝true、更新後＝false
* aria-atomic : 更新後に更新した範囲だけを含んでたらfalse、ほかのも含んでいて一部だけの更新だったならtrue
* aria-live : 更新されても伝達しない＝off、適切そうなタイミングで伝える＝polite、即座に伝える＝assertive

ドラッグ＆ドロップするもの・ドロップ先
* aria-grabbed : ドラッグ＆ドロップできない＝undefined(デフォルト)、ドラッグ＆ドロップできるけどまだされてない＝false、ドラッグ＆ドロップできるし今されてる＝true
* aria-dropeffect : ドロップされたものを複製する＝copy、ドロップされたものをされた先に移動する＝move、ほかいろいろ
  * http://www.webcreativepark.net/html/wai-aria/aria-dropeffect/

ラベルと実体
* aria-label="ラベルとなるワード"
* 実体に充てるラベルの要素を指定する aria-labelledby

タブ部品：role="tablist" ＋ role="tab" ＋ role="tabpanel"
* パネル[role="tabpanel"] のトリガーになるタブを `aria-labelledby="トリガーのタブ"` で指定
* トリガー[role="tab"] によるトリガーでアクティブにするパネルを `aria-controls="対応するパネル"` で指定
* アクティブになっているパネルのトリガー : aria-selected="true"
* そのトリガーでパネルが開いている : aria-expanded="true"
* トリガーを複数アクティブにできる : aria-multiselectable="true"

フォーム部品：
* 入力必須のinput : aria-required
  * このariaで示せてるので、入力必須であることを示す物理的記号には aria-hdden="true" で覆える
* aria-multiline : 複数行テキスト入力できる＝true、できない＝false
* aria-invalid : submitされたformにおいて、不正な値を入力されたフォームに設定する
※ formタグにnovalidate : 妥当性のチェック（入力内容の検証）を無効にする
* aria-errormessage : エラー文言を入れる要素を指定する
* aria-checked : true / false
* aria-disabled : true / false

  ※ チェックボックス：role="checkbox"

パンくず・現在地：
* aria-current : 現在地である＝true

？：
* aria-orientation : 水平＝horizontal、垂直＝vertical
* aria-level : 階層の高さを数値で
* aria-hidden : ユーザエージェントに認識させたくない＝true
* aria-haspopup : ポップアップメニューがある＝true
* aria-flowto : 次の読み上げ対象を指定
* aria-describedby : 解説している要素を指定
