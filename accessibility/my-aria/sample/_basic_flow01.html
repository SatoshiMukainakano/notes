方針

header / footer / main

「body以外のセクションに属さない header 、body以外のセクションに属さない footer 、階層的に正しい main でそのページの必要分を構成する」を前提としたタグ充てをする

このとき、webサイトの広範で共通の内容を備えている header 部分は role="banner"、同じく footer 部分は role="contentinfo" を帯びる。

解説

前段

このHTMLそのものを見れば、当標本で説明したかったことをいくらか確認できる。

header / footer / main

「body以外のセクションに属している header」は、属するセクションにおける header であって role="banner" も帯びない。

「body以外のセクションに属している footer」は、属するセクションにおける footer であって role="contentinfo" も帯びない。

「階層的に正しくない main」は、そもそも不適切。25日

補助

厳密には、「headerタグは webサイトのヘッダ部品だから充てられている」、という解釈には誤解が含まれる。いわゆるヘッダ部品じゃない箇所でも利用に適した用法があり、headerタグ の先祖要素・文書全体を見て解釈を掴むべきである。

厳密には、footerタグも 前述の header タグと同様。

セクショニング

article 1つの中で、コンテンツの主題・主旨が1セットになっていれさえすれば、同一文書の中に article が複数あってもよい。

このとき、webページにおけるいわゆるヘッダ・フッタ（：どのページでも同一的に表示・機能させたいもの）は、どのarticleにも含ませないで考える。
